## Security & correctness (do first — audit findings)
- [ ] **P0 segfault**: `_qid_passive_extract` (interlang.py) reads the shared pooled
      Archive with no `_zim_lock` on every article load. Fix: dedicated
      `open_archive()` handle (mirror `_build_qid_index`). Confirmed by audit.
- [ ] **Test the segfault class**: commit one tiny real fixture `.zim` (libzim
      Creator, ~3 articles); add an N-thread stress test on `/read`+`/search`
      same ZIM. Nothing today catches a lock-drop.
- [ ] **/dl/ public exposure (LIVE)**: host-mode container + bridge cloudflared →
      Zimi sees the bridge gateway (private) as the client for ALL WAN traffic →
      `/dl/` serves whole ZIMs to the internet. Fix: when the hop is a trusted
      proxy but no real client IP is forwarded, FAIL CLOSED (public), and honor
      `CF-Connecting-IP` as the un-spoofable real-IP source. Also fixes the XFF
      leftmost-trust issue in the same pass.
- [ ] P1: pooled-archive lock assigned after archive (search.py) → silent
      search-result drop under load. Assign lock first / store as tuple.
- [ ] P1: `get_article_languages` caps content reads under the global lock
      (~5, not 30) → no server-wide stall.
- [ ] mDNS advertises `p2p.get_bt_port()` not the raw env (server.py).
- [ ] `_parse_range` guards malformed ranges (http.py) → no 500 on `bytes=abc-`.
